# Security Policy

## Supported Versions

We release patches for security vulnerabilities. Currently supported versions:

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability, please do the following:

1. **Do not** open a public issue
2. Email the details to: security@example.com (replace with actual email)
3. Include the following information:
   - Type of vulnerability
   - Full paths of source file(s) related to the vulnerability
   - Location of the affected source code (tag/branch/commit or direct URL)
   - Step-by-step instructions to reproduce the issue
   - Proof-of-concept or exploit code (if possible)
   - Impact of the issue, including how an attacker might exploit it

## Response Timeline

- We will acknowledge receipt of your vulnerability report within 48 hours
- We will provide a detailed response within 7 days
- We will work on a fix and keep you informed of progress
- Once the vulnerability is fixed, we will publicly disclose it

## Smart Contract Security

For smart contract vulnerabilities:

1. Report immediately if you find any issues
2. Do not exploit the vulnerability
3. Do not share the vulnerability publicly until it's fixed
4. We may offer a bug bounty for critical vulnerabilities

## Best Practices

When using this application:

- Never share your private keys
- Always verify contract addresses on BaseScan
- Use hardware wallets for large amounts
- Keep your software up to date
- Be cautious of phishing attempts

Thank you for helping keep Base WalletConnect and our users safe!
